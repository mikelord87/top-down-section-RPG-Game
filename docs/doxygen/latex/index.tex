A small top-\/down action RPG prototype. The player controls a knight character, explores a 2D world, and interacts with enemies and props.

\hypertarget{index_autotoc_md2}{}\doxysection{\texorpdfstring{Features}{Features}}\label{index_autotoc_md2}

\begin{DoxyItemize}
\item Top-\/down player movement (WASD)
\item Player melee weapon (sword) with a weapon collision rectangle
\item Basic health / alive state handling
\item Enemies with damage interactions
\item Props / world objects
\item Doxygen API documentation from source comments
\end{DoxyItemize}

\hypertarget{index_autotoc_md4}{}\doxysection{\texorpdfstring{Project Architecture}{Project Architecture}}\label{index_autotoc_md4}
This project is organized around a few core classes\+:


\begin{DoxyItemize}
\item {\ttfamily \doxylink{class_base_character}{Base\+Character}} ~\newline
 Shared functionality for characters (common data/state and behavior used by both player and enemies).
\item {\ttfamily \doxylink{class_character}{Character}} ~\newline
 The {\bfseries{player-\/controlled}} knight. Handles input, movement updates, weapon drawing/collision, and player health.
\item {\ttfamily \doxylink{class_enemy}{Enemy}} ~\newline
 AI-\/controlled character logic (movement/behavior and interactions with the player).
\item {\ttfamily \doxylink{class_prop}{Prop}} ~\newline
 Non-\/character world objects (environment props, interactables, decorations, etc.).
\end{DoxyItemize}\hypertarget{index_autotoc_md5}{}\doxysubsection{\texorpdfstring{Key gameplay data (examples)}{Key gameplay data (examples)}}\label{index_autotoc_md5}

\begin{DoxyItemize}
\item Player health is initialized from {\ttfamily CHARACTER\+\_\+\+PLAYER\+\_\+\+HEALTH} (see {\ttfamily \doxylink{_character_8h}{Character.\+h}}).
\item Weapon sprite path is defined in {\ttfamily \doxylink{_character_8h}{Character.\+h}} (e.\+g., {\ttfamily KNIGHT\+\_\+\+WEAPON\+\_\+\+SWORD\+\_\+\+PATH}).
\end{DoxyItemize}
