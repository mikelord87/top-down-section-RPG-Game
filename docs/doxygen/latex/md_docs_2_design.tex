\hypertarget{md_docs_2_design_autotoc_md105}{}\doxysection{\texorpdfstring{Game Concept}{Game Concept}}\label{md_docs_2_design_autotoc_md105}
A top-\/down action RPG prototype where the player controls a knight ({\ttfamily \doxylink{class_character}{Character}}) exploring a 2D world. The player encounters enemies ({\ttfamily \doxylink{class_enemy}{Enemy}}) and interacts with world objects ({\ttfamily \doxylink{class_prop}{Prop}}).

\DoxyHorRuler{0}
\hypertarget{md_docs_2_design_autotoc_md107}{}\doxysection{\texorpdfstring{Core Pillars}{Core Pillars}}\label{md_docs_2_design_autotoc_md107}

\begin{DoxyItemize}
\item {\bfseries{Readable movement and combat}}\+: simple controls, clear hit feedback
\item {\bfseries{Fast iteration}}\+: add small features frequently
\item {\bfseries{Data-\/driven growth}}\+: move constants and tuning values to one place as the game scales
\item {\bfseries{Maintainable architecture}}\+: shared behavior in {\ttfamily \doxylink{class_base_character}{Base\+Character}}, specialized behavior in derived classes
\end{DoxyItemize}

Typical approach\+:
\begin{DoxyItemize}
\item Keep player on screen center
\item Draw enemies/props relative to player world position offset
\end{DoxyItemize}

Notes\+:
\begin{DoxyItemize}
\item Distinguish {\bfseries{world space}} (true positions) vs {\bfseries{screen space}} (draw positions)
\item Keep conversions consistent (one function or module)
\end{DoxyItemize}
